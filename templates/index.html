▶ Как это работает

Что это

Прототип ИИ-агента для мониторинга IT-инфраструктуры. В отличие от обычного чат-бота, агент сам обращается к инструментам — запрашивает Zabbix API, выбирает нужное действие и формирует ответ на основе реальных данных, а не обученных знаний.

Ветки действий

СТАТУС СЕРВЕРА «что с report3?» → ищет хост по имени, IP или номеру КЭ, получает активные триггеры, запускает экспертный разбор через LLM.
СТАТУС СИСТЕМЫ «как биллинг?» → ищет группу Zabbix, собирает триггеры по всем хостам, даёт сводный анализ.
СПИСОК СЕРВЕРОВ «какие серверы в казначействе?» → перечисляет хосты группы, отмечает у каких есть проблемы.
СПИСОК МЕТРИК «что мониторится на bukh1?» → возвращает все метрики хоста напрямую из API. Клик на числовую метрику строит график.
ГРАФИК «покажи память на report5» → LLM выбирает нужную метрику из полного списка хоста, строит временной ряд за 3 часа.
ДИАЛОГ Если сервер или система не упомянуты — отвечает в контексте предыдущего объекта разговора или просит уточнить.

Данные

Агент не угадывает и не выдумывает. Вся информация берётся напрямую из Zabbix — если сервер не найден в мониторинге, агент честно об этом скажет. Если название системы совпадает с несколькими группами, уточнит, какую именно вы имеете в виду.

Модели

Переключатель в шапке: Pro — оптимальный баланс, Max — точнее и подробнее, Lite — быстрее всего. История разговора сохраняется при смене модели.

Промпты ⚙
